嵌入式boot

嵌入式开发 2020-01-06 1735 字 4 分钟

虽然嵌入式开发大多时候需求都是稳定的，但是不可避免在发布之后仍然需要修复一些bug及增加一些新功能。

故在空间允许的情况下，我一般都建议每个工程师都在项目中增加Boot.

相对于linux常用的uboot，一般嵌入式常用的boot则较为简陋，但是只是适用性的差别，大多基本功能都是相同的。

下文中所指的boot 都代指常用mcu开发中的boot，非内核引导的boot(如uboot)。

boot的作用

列了boot的主要的一些功能，下面将逐一说明各个功能的作用及原理。

  1. 负责引导App(核心功能)

    引导App是boot存在的意义，此功能是核心功能，但boot也可脱离app的情况下独自允许，但是app却不能脱离boot存在。可见boot在整个系统中是非常关键。 所以一般boot经发布后较少升级(boot也可以升级)，避免引入新bug，并且boot本身功能较为单一，在开发时已经确定，后期迭代的可能性较少。 Boot引导App的方式也很简单，大多数嵌入式开发，只需要将pc指针强行指向App中断向量表即可。

    以ARM-M core为例，正常中断向量表第一个u32为sp指针，第二个为reset中断指针，所以在开发的时候只需要将app的中断向量表固定到app flash地址空间内即可。

    当然一般默认为flash的起始，当然也可以指向RAM。只要不存在冲突中断向量表是可以随便指向任何位置(看内核手册，看地址对齐要求)。

    比如想在运行时动态修改中断向量表，则必须将中断向量表copy到RAM然后去修改。本质上只需理解中断向量表为一个函数指针数组，内核会根据中断编号在表中索引函数指针然后调用。

    以ARM-M core为例，中断向量表配置寄存器为 SCB->VTOR

    PS:其他linux等，本质上模型都是与mcu一样，只不过存在很多的优化或者软件上的策略。

  2. 校验App完整性

    上面咱们说了，boot主要任务就是引导app，但是这个app并非可靠的，可能因为读取错误，存储错误，亦或者就是工程师原因等等，造成这个app并不能正常运行。 这个时候boot就是最后一道防线。

    app的错误大致可以分为两类 代码损坏， 逻辑错误。

    代码损坏 可能是从工程师到写入芯片内中间环节的问题，也可能是辐射，非易失器件本身特性，传输等等。都会造成app代码损坏。 可以在编译之后再代码中加入校验，比如再bin文件中增加校验信息，boot引导之前先计算校验是否正确，等等。

    逻辑错误 非定义的外部输入，逻辑不完善等等，造成程序执行混乱。这个时候建议开启开门狗，然后再boot里面判断看门狗是否苏醒。 正常逻辑，看门狗是永远不会苏醒，一旦醒了，就说明逻辑存在问题，这个时候可以进入保护逻辑，或者其他措施。

    App完整性校验目的就是通过各种手段判断app是否可靠，无论是引导前还是引导后，只是手段不同。

  3. 更新App 如果产品存在升级功能，则需要更新将新固件下载到芯片中。 升级也很简单，这部分不过就是通讯口的收发，非易失器件擦写，不过做好考虑的也很多。 A. 通讯口传输新固件，大多都是切片传输，每帧增加校验，校验错误则重传 B. 传输一半中断或者断电咋办？ 空间足够则采用缓冲，先保存新固件，校验完整后，再copy到app区域

  4. 硬件初始化 boot也是一个功能相对稳定的固件，本质也需要初始化硬件，只是对硬件要求较低。

    1.初始化时钟 2.初始化内存,一般内存要求较低，内部有SRAM就省的自己初始化了 3.关闭看门狗 4.初始化通讯口(如果存在更新App)

  5. 故障恢复

    这部分起始就是完整性校验后续的工作，发现app存在故障该如何解决，比如回退到上版本，跳转到故障检测固件，还是直接停留再boot等待排查故障等等。

代码例程(针对ARM-M)

  1. 跳转代码(boot->app)

    #define USER_APP_FLASH_START_ADDR   0x08008000
typedef void (*pFunction)(void);
void Run_User_Application(void)
{
    uint32_t JumpAddress = *(__IO uint32_t*)(USER_APP_FLASH_START_ADDR + 4);
    pFunction Jump_User_Application = (pFunction)JumpAddress;
    RCC_HSICmd(ENABLE); //切换到内部HSI
    __disable_irq();
    NVIC->ICER[0] = 0xffffffff;//关闭所有中断
    NVIC->ICER[1] = 0xffffffff;
    RCC_SYSCLKConfig(RCC_SYSCLKSource_HSI);//HSI
    while(RCC_SYSCLKSource_HSI != RCC_GetSYSCLKSource());//等待时钟稳定
    __set_MSP(*(__IO uint32_t*)USER_APP_FLASH_START_ADDR);
    Jump_User_Application();
}
